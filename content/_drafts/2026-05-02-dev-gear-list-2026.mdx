## 为什么整理这份清单

每年五月我都会盘点一下自己的开发装备——哪些还在吃灰，哪些真的每天都在用，哪些花得不值。今年尤其想写这篇，因为过去半年工具链变化太大了：AI 编码工具彻底重构了我的工作流，同时也在持续烧钱。

这篇文章不是带货，纯粹是记录我自己真实的日常使用组合。有踩过的坑，也有花得值的地方。如果你正在搭建自己的开发环境，或者想对比一下别人的装备，希望这份清单能给你一些参考。

先说结论：**2026 年做开发，纯软件订阅一年能轻松花掉上万，硬件一次性投入大但能用好几年。** 下面按类别细说。

---

## 开发工具

这块是我花钱最多的地方，也是变化最频繁的领域。2026 年，AI 辅助编码已经从"加分项"变成"基础设施"了，但坑也来了——工具越来越多，功能互相重叠，订阅费也在叠加。

### AI 编码工具三件套

目前我同时用着三个 AI 编码工具，不是我有选择困难症，而是它们各自解决不同的场景：

| 工具 | 月费（USD） | 月费（约 ¥） | 主要用途 | 使用频率 |
|---|---|---|---|---|
| Cursor | $20 | ¥145 | 日常编码主力 IDE | 每天 |
| Claude Code | $200 | ¥1,450 | 复杂重构 / 跨文件任务 | 每周 3-5 次 |
| GitHub Copilot | $10 | ¥72 | 补全提示 / 备用 | 每天（被动） |

**Cursor** 是我的主力编辑器，从 VS Code 转过来已经快两年了。2026 年的版本在代码索引和上下文理解上又提升了不少，Agent 模式处理中等难度的 feature 基本一次成功。最大的痛点是偶尔会"幻觉"引用不存在的 API，需要人工复查。¥145/月的价格在我接受范围内，毕竟它直接替代了我以前花在补全插件和搜索引擎上的时间。

**Claude Code** 是企业版订阅，价格确实高，但复杂任务上碾压其他工具。我主要用它做跨文件重构、数据库迁移脚本生成、以及写测试。它的优势在于对整个代码库的理解深度——不是单文件级别的补全，而是真正能把握架构层面的改动。缺点也很明显：$200/月对个人开发者来说不便宜，而且本地执行速度偶尔不稳定。我的建议是：如果你每周都有几次超过 200 行代码的改动需求，这个钱值得花；如果日常只是修修补补，Cursor 就够用了。

**GitHub Copilot** 现在更多是作为"兜底"存在。它的行级补全依然好用，但在 agent 能力上被 Cursor 和 Claude Code 甩开了。留着主要是因为在 GitHub 生态里集成度最好，review PR 的时候能自动给出建议。$10/月的价格几乎没有存在感，算是性价比最高的一个。

### 终端

**Warp** 每月 $15，约 ¥108。坦白说，我对 Warp 的感情很复杂。它的 AI 命令建议和团队共享命令功能确实好用，但性能上偶尔有卡顿，尤其是打开大型项目的时候。如果你习惯了 zsh + 自定义配置的顺滑，Warp 可能需要一点适应期。我现在用它的核心原因是 AI 命令提示——遇到不熟悉的 CLI 工具时，自然语言描述就能生成命令，比翻文档快太多。

如果预算有限，**iTerm2 + Oh My Zsh** 的组合完全免费，配合一些 alias 和插件也能达到 80% 的效率。Warp 的溢价主要在于开箱即用的 AI 能力。

## 效率工具

效率工具这块我走的是"够用就好"路线，不追求全家桶。

### Notion（$10/月，约 ¥72）

我主要用来做项目管理和知识库。2026 年的 Notion 在 AI 方面加了不少功能，比如自动摘要、数据库智能过滤，但我基本不用——我的使用场景很简单：记录会议笔记、跟踪项目进度、维护个人 wiki。免费版的功能其实对单人也够了，但 $10/月的 Plus 版能解锁上传大文件和版本历史，对我这种会存大量截图和设计稿的人来说是刚需。

### Obsidian（免费）

本地笔记的终极选择。我把它作为 Notion 的补充——Notion 管项目和协作，Obsidian 管个人思考和知识整理。双向链接和插件生态是我离不开它的理由。2026 年 Obsidian 的 Canvas 功能已经很好用了，我偶尔用来画架构草图。唯一的缺点是多设备同步需要自己搞，我用 iCloud 解决，基本没遇到过冲突。

### Raycast Pro（$8/月，约 ¥58）

macOS 上最好用的启动器，没有之一。免费版已经覆盖 90% 的场景（应用启动、窗口管理、剪贴板历史），但我开了 Pro 主要是为了 AI 功能——直接在 Raycast 里问问题、生成代码片段、转换数据格式。它把原本需要切换到浏览器的操作省掉了，日积月累能省不少时间。

### Arc 浏览器（免费）

用了两年 Arc，最大的感受是 Spaces 功能确实改变了浏览器的工作流。工作、个人、学习各一个 Space，互不干扰。2026 年 Arc 在资源占用上优化了不少，不像早期版本那样吃内存。缺点是和某些国内网站的兼容性偶尔有问题，遇到打不开的页面我会切回 Chrome。

## 硬件装备

硬件是一次性投入，我的原则是：**显示器和键盘不要省，其他看预算。**

### MacBook Pro M4 Max（64GB）—— ¥22,999

2025 年底购入，用了快半年。M4 Max 的性能对于我的使用场景来说完全过剩——本地跑大语言模型编译、同时开几十个 Docker 容器、编译大型前端项目，基本没有卡顿的体感。64GB 内存是我最不后悔的升级选项，本地跑 7B/14B 模型做开发调试的时候，32GB 明显不够用。

如果你主要在云上开发（GitHub Codespaces / 远程服务器），那 M3/M4 芯片的 Air 也够用，能省下一大笔钱。但我的习惯是本地能跑的就跑本地，所以 Max 芯片 + 大内存对我来说是刚需。

### Keychron Q1 Max 机械键盘 —— ¥899

2026 年我用过的性价比最高的键盘。Gasket 结构、热插拔、蓝牙/2.4G/有线三模，该有的都有了。我用的是红轴，敲代码手感不错，长时间写代码手指不累。Q1 Max 的重量适中，放在桌上不飘。¥899 的价格在这个配置下基本没有对手。之前用过 HHKB，手感是好但价格差了一倍多，日常使用体验上 Q1 Max 已经达到 90% 的水平了。

### LG 4K 显示器 —— ¥2,599

27 英寸 4K IPS，色彩还原好，长时间看代码不刺眼。这块屏幕我用了三年，至今没有坏点。选显示器我的经验是：分辨率优先 4K，刷新率 60Hz 足够（不做游戏），面板选 IPS 或 OLED。¥2,599 的价格算是中档，但这个尺寸和分辨率下性价比不错。

### Ergotron 显示器支架 —— ¥1,299

这个可能是清单里最被低估的工具。显示器支架解放了桌面空间，更重要的是能随意调整高度和角度，对颈椎友好多了。Ergotron 的气弹簧调节很顺滑，单手就能拖动显示器。用了之后才知道以前把显示器直接放在桌面上有多憋屈。¥1,299 不便宜，但健康投资不能省。

## 年度花费总结

把上面的订阅费加总，再看看整体花费：

| 类别 | 月费（约 ¥） | 年费（约 ¥） | 说明 |
|---|---|---|---|
| 开发工具订阅 | ¥1,775 | ¥21,300 | Cursor + Claude Code + Copilot + Warp |
| 效率工具订阅 | ¥130 | ¥1,560 | Notion + Raycast Pro |
| **软件订阅合计** | **¥1,905** | **¥22,860** | 每月一杯星巴克的钱变成了开发工具 |
| 硬件装备 | — | ¥27,796 | 一次性投入（按购入价） |

如果按五年折旧算硬件，每年的综合成本大约是：

| 项目 | 年度成本（¥） |
|---|---|
| 软件订阅 | ¥22,860 |
| 硬件折旧（5年） | ¥5,559 |
| **年度总成本** | **¥28,419** |
| **月度均摊** | **¥2,368** |

说实话，这个花费比我年初预期的高了不少。主要原因是 Claude Code 的企业版订阅占了大头。但回顾这一年，工具带来的效率提升是实打实的——以前需要一整天的跨文件重构，现在 Claude Code 半小时就能完成，人工 review 一下就行。

### 购买建议

1. **先理清需求再花钱**：别被"AI 编码革命"的营销裹挟。先用好免费/低价工具，确定瓶颈了再升级。
2. **硬件优先投显示器和键盘**：这两样直接影响每天的使用体验，值得花钱。电脑看预算，够用就行。
3. **订阅定期review**：每季度检查一次订阅使用情况，用不上的果断停掉。我的 Claude Code 如果不是每周高频使用，早就降级了。
4. **别忽视免费工具的力量**：Obsidian、VS Code、Arc 浏览器，这些免费工具的质量已经很高了，不一定什么都要付费版。
